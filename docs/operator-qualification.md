# Operator Qualification

Operator qualification in Harvest OS is an internal operating readiness workflow. It connects training progress, assessment attempts, practical evaluation, credential evidence, aircraft model authorization, payload authorization, and job-specific assignment gates.

## Qualification Inputs

- TrainingCourse and TrainingEnrollment: Harvest-authored training progress and completion.
- PracticalEvaluation: supervised field evaluation and lead/chief pilot signoff.
- OperatorCredential: tracked and verified credential evidence.
- OperatorQualification: Harvest-qualified scope for model, payload, state, and operation type.
- ReadinessCheck: job-specific result for an operator, aircraft, and application job.

## RDO Demo Roles

- Lead Operator: can be ready for assignment when all Harvest gates pass.
- Operator trainee: can be blocked while training, practical signoff, and state credential evidence remain incomplete.
- Backup Operator: can be blocked by expired credentials or job-specific state/payload/aircraft requirements.

## Required Language

Say:

- credential tracked
- credential verified
- Harvest-qualified
- ready under Harvest rules
- practical signoff complete

Do not say Harvest issues FAA, state, pesticide, or manufacturer certifications.

Official Hylio training content should be represented as placeholder links or evidence fields. Harvest-authored content should focus on the operating workflow, readiness expectations, SOP execution, evidence capture, and recordkeeping.
