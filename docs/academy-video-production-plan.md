# Harvest Drone OS Academy Video Map

The Academy renders a real lesson video from each lesson's `videoUrl`. Generated WebM files live under `public/training-videos/academy/generated/` during development and can later be replaced by hosted MP4/HLS URLs without changing the Academy UI.

## Where Videos Fit

- `shared/academy.js` is the source of lesson metadata. Add or update `lesson.videoUrl` there.
- `src/pages/AcademyPage.jsx` renders `.webm` and `.mp4` URLs with a native `<video controls>` player.
- `src/pages/AdminAcademyPage.jsx` already exposes a video URL field for admin editing.
- HyperFrames or other composition sources can live beside the local prototypes, but the training gate should point to rendered MP4s.

## Production Notes

- Current generated videos are short demo briefings. Expand each into 60-120 second operator videos later.
- Use the local `public/training-videos/academy/DESIGN.md` visual identity for all HyperFrames compositions.
- Script each video around one operator action: watch, read, open resource, complete checklist, advance.
- Regenerate local video assets with `python scripts/generate-academy-videos.py` after updating the lesson script map.
- Every lesson should also have a verification prompt so completion proves the pilot watched the briefing, reviewed the written content, and passed a knowledge check.
